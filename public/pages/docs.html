Overview

Documentation that feels fast, technical, and premium.

SandboxCodex is an AI-powered platform for automated code review, testing, and PR generation. Use this guide to install the CLI, connect a repository, configure agents, and ship reliable changes from an isolated sandbox.

01

Agent workflows

Run code review, tests, changelog generation, and PR preparation from one command surface.

02

Safe sandboxing

Keep secrets, environment variables, and repository changes scoped to the workspace.

03

Team ready

Standardize setup through config files and reusable templates for every project.

Install

Installation

Install the CLI globally or add it to an existing project so every contributor can run the same commands.

terminal
npm install sandboxcodex
npx sandboxcodex --version
Quickstart

Launch your first workspace

Create a project, enter the directory, and start the development server. The initializer creates a baseline configuration, sample automation, and a local dashboard.

1

Create a sandbox workspace with a production-ready folder structure.

2

Connect GitHub and choose the workflows your repository should run.

3

Start the dashboard and monitor generated checks in real time.

terminal
sandboxcodex init my-project
cd my-project
npm start
Configuration

Configure your automation

Use sandbox.config.js to define runtime, repository access, and default agent behavior.

sandbox.config.js
module.exports = {
  projectName: 'my-project',
  sandbox: {
    runtime: 'nodejs',
    checks: ['lint', 'test', 'build']
  },
  github: {
    token: process.env.GITHUB_TOKEN,
    defaultBranch: 'main'
  }
};
